The purpose of this website is summarized below.

o Web Site and Target Audience o

Just as stated on the main page, the goal of this website is to provide a system that ranks and filters applicants based on how a company wants to filter. Users have been inputted already to provide a basis of rankings. The unique part of the filtration system is that companies can provide different weights to bias filtration based on which categories are deemed more important. Usually, companies do this inherently without any numerical basis - it is more heuristical in nature. By quanitfying this we allow the company to focus on further rounds for interviews versus the preliminary rounds. The target audience is the sponsor companies who will have this website/platform as a link on their applications. The other audience is of course the applicants. Feedback will be faster based on company cutoffs so applicants do not need to wait as long to hear back. Seemingly biased, but this is inherent in most job application processes based on company needs and values at the time.

o Design Considerations and Aesthetics o

At first I considered having a logo because I wanted the site to be memorable, but then it turned out that the logo got hidden in all of the forms and procedures. Instead I thought of having a company "symbol" - here its the cloudy sky. This is memorable especially with the company name as it is in the background of every page. I wanted this to have a spacey feeling where concentration is mostly on the forms. I thought that having most of the information centered in the middle would allow for more focus and a bigger view of the sky in the background. I thought about how all websites with ads have them placed on the sides. We have been conditioned to ignore ads in a fast manner, and we end up focusing on the middle of the page. So this can go two ways: 1) allows a view of the sky - company symbol and 2) if sold to companies can allow easy space for advertisements to go.

o Client-Side User Interactivity o

The main user event is the taking of the test. This is given weight and focus due to several forms on the page, but more importantly creates a fast-paced environment with a timer. The user inherently wants to get a low time, so this provides a basis for interaction. Furthermore, on the main student registration page there is a table for job positions to choose from that has been generated with a script - this then populates the job in the forms that the user wants to apply to. Lastly, in a similar manner a script generates a tips menu that a user can push buttons with to go through several tips - forwards and backwards.

o Information Exchange with the Server o

The main server side processing is when we are transferring information from one page to another. This is mainly the information from input forms such as username and email. Populating forms dynamically happens through echos of PHP when transferring data from one page to another. Session Storage was considered, but consolidating variables into one array seemed to be overkill when we could just extract it from the URL. Each ranking filtering does indeed use server side processing to get information from the mySQL tables/lists. On the main page we also validate the input for the company log in by comparing the password and username with what we have in the database.

o Operations Performed on the Server o

The biggest operations performed are those within the ranking and listing ranks inside of the table in the analytics2 file. Each weight given - and obtained through GET - is multiplied by the values in the mySQL database already. This then obtains an overall weighted score value. Values are also listed in ascending in descending order based on user categories selected for ranking. Similarly, depending on which job role is chosen, and which company is logged in the processing works in that manner.
